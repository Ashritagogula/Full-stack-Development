🧠 CSS Combinators & Pseudo-Classes (Real Life Explanation)

🔗 1. CSS **Combinators**

Combinators help define **relationships between HTML elements**, like parent-child or siblings. Think of it like **family relations**!

1.1 A B → Descendant Selector

Selects all B inside A, no matter how deep.

Think of it like: "All kids living inside a house A." 
div p {
  color: green;
}

1.2 A > B → Child Selector

Selects only the direct child B of A.

Real-life: "Only the children (not grandkids) of Parent A." 
.parent > .child {
  color: blue;
}

1.3 A + B → Adjacent Sibling

Selects element B that comes just **after A**.

Like: "A girl sitting right next to you in class." 
.direct-next + p {
  color: orange;
}

1.4 A ~ B → General Sibling

Selects all B that are siblings of A **and come after** it.

Like: "All younger siblings after an elder brother." 
.sibling ~ p {
  color: purple;
}

🎭 2. CSS **Pseudo-Classes**

Pseudo-classes target an element based on its **state** or **position** (like when you hover, click, or focus).

2.1 :hover – When you hover

Hover over this box
.hover-box:hover {
  background-color: skyblue;
}

2.2 :active – When clicked

.click-me:active {
  background: green;
  color: white;
}

2.3 :focus – Input selected

input:focus {
  border: 2px solid blue;
}

2.4 :nth-child(n) – Select by position

  • Item 1
  • Item 2
  • Item 3
  • Item 4
.list li:nth-child(even) {
  color: red;
}
